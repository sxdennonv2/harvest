Excel to a High Standard · Lesson 0010

XLOOKUP: pull matching details

Tied to your mission: you have an order list with just product names, and a separate catalogue that knows each product's supplier and case cost. XLOOKUP pulls the right detail across from the catalogue for every line — so you stop copy-pasting supplier names by hand and start joining your data like the analyst leadership trusts with the numbers.

Practice workbook · open this first

Open the course workbook and click the XLOOKUP tab — a product catalogue sits on the left (columns A–C) and an order list on the right (column F) with blank Supplier and Case-cost columns to fill.

The one idea

XLOOKUP takes a value, finds it in one column, and returns the matching value from another. You read it as: look up this, in that column, and return the value from this other column. The shape is =XLOOKUP(lookup_value, lookup_array, return_array).

To find a product's supplier: =XLOOKUP(F2, $A$2:$A$11, $B$2:$B$11). Swap the return column to $C$2:$C$11 and you get the case cost instead. Unlike the old VLOOKUP, it doesn't care whether the answer sits left or right of the key, and it never breaks when you insert a column.

Why it matters to a specialist analyst

Your working data is scattered across systems and extracts — an order list here, a supplier catalogue there, a cost file somewhere else. XLOOKUP is how you stitch them into one honest row: product → supplier → cost → margin, all pulled, never retyped. That row is what turns into the summary or chart that lands on your director's desk, and a retyped figure is where a compliance or sustainability metric quietly goes wrong. A lookup is where that error dies before it ever reaches the report.

The bad habit this undoes Eyeballing the catalogue and typing each supplier and cost next to the order by hand — slow, and quietly wrong the moment a cost changes. And its near-cousin: an old VLOOKUP with a hard-coded column number that silently returns the wrong column after someone inserts a field. XLOOKUP names the return column, so it can't drift.

How to do it

On the XLOOKUP tab, in G2 type =XLOOKUP(F2, $A$2:$A$11, $B$2:$B$11) to pull the supplier, and in H2 type =XLOOKUP(F2, $A$2:$A$11, $C$2:$C$11) for the case cost. Fill both down. Lock the catalogue ranges with $ so every order row searches the whole catalogue.

Handle a miss gracefully: add a fourth argument for when nothing matches — =XLOOKUP(F2, $A$2:$A$11, $B$2:$B$11, "not found") — so a typo shows a clear word instead of a bare #N/A.
