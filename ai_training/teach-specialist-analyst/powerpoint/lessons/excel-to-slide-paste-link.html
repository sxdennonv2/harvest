PowerPoint to a High Standard · Lesson 0005

Excel to slide: paste-link, never screenshot

Tied to your mission: your director's monthly review runs on a chart that has to keep telling the truth after you've closed the workbook and moved on to next week's numbers. This lesson closes a loop the Excel course opened — that course taught you to build a chart that says one thing cleanly; this one teaches you to get that chart onto a slide without cutting the wire back to the numbers it's reporting.

Practice pack · open this first

Open the practice pack and find Scenario B — The screenshot deck: the monthly review deck carries four "charts", all screenshots of the course workbook, and the workbook's revenue figures changed yesterday. The deck didn't notice.

The one idea

A screenshot of a chart is a picture of data at one moment — it freezes a lie the day the workbook moves. Paste-link keeps the slide's chart alive to its workbook instead: change the number in Excel, the deck follows, with no one having to remember to redo the slide. The choice of paste flavour is the actual skill. A linked chart updates but needs the workbook nearby to refresh from. An embedded chart is self-contained and heavier — a snapshot that still carries its editing tools, but won't move when the source workbook does. A picture is dead on arrival, and that's fine — but only for things that were never data in the first place.

Monthly decks — the ones that get rebuilt against the same workbook every cycle — want linked. Decks that travel, that leave your machine for someone else's laptop or an external inbox, want the link severed deliberately at export, not accidentally the moment you pasted. Choosing wrong in either direction is how a number goes stale without anyone deciding it should.

Why it matters to a specialist analyst

Your sustainability, category and compliance decks run on the same handful of charts every cycle, and the SAP extract or dashboard pull behind them keeps moving — a restated figure, a corrected volume, a recut by category — right up until the morning of the review. A linked chart absorbs that change for free: open the deck, let it refresh, and the slide already agrees with the extract. A screenshot absorbs nothing. It sits there looking finished while the number underneath it quietly goes wrong, and the first person to notice is usually leadership reading last cycle's figure aloud as if it were this cycle's.

The bad habit this undoes Alt-PrtScn as the Excel-to-slide pipeline: build the chart in Excel, screenshot it, paste the picture into the deck, repeat for every chart, every month. Four screenshots a cycle, each one a stale number waiting for a director to read it aloud — and nothing about the slide tells you it's stale, because a picture never complains. The habit survives because it "works" every time you paste it; it only fails the one time the workbook moves after you've stopped looking, which is every month, eventually.

How to do it

Rebuild Scenario B's revenue-by-category chart as the pack's box describes. In the workbook, select the existing revenue chart and copy it (Ctrl+C on the chart object, not the underlying cells). Switch to the deck, click the slide where the screenshot currently sits, delete the picture, then Home → Paste → Paste Special → Paste Link (in the Paste Special dialog, choose Microsoft Excel Chart Object with the Paste Link option selected, not the default plain paste). That one dialog is the whole decision this lesson is about — picking Paste Link instead of the default embed is what keeps the slide wired to the workbook rather than just carrying a heavier copy of it.

Then run the proof, because a linked chart that's never been tested is a claim, not a fact: go back to the workbook, change one revenue number that feeds the chart, save it, then return to the deck and right-click the chart to Update Link (or close and reopen the deck, which refreshes links automatically). Watch the column actually move. If it doesn't, you pasted an embed or a picture, not a link — go back and redo the Paste Special.

Once it's live, apply the Excel course's chart discipline on the slide itself, the same as you did in the workbook: one comparison per chart, gridlines kept light, and no legend sitting beside a single data series. A linked chart that's still cluttered with Excel's defaults hasn't finished the job — the link keeps the numbers honest, but the cleanup is what keeps the slide readable.

Check yourself

Answer these before moving on — immediate feedback below each.

1. A revenue figure changes in the workbook the morning of the review. Which slide chart reflects that change without you touching the deck?

2. A deck is about to leave the building — going to an external recipient whose machine won't have your workbook open. What should happen to its linked charts before you send it?

3. A single-series revenue-by-category chart, now correctly linked, still shows Excel's default legend and heavy gridlines. Is the job done?

Try it for real

On Scenario B's revenue chart, do the full rebuild: copy the chart in the workbook, Paste Special → Paste Link it into the deck in place of the screenshot, then clean it to the Excel course's discipline (one comparison, light gridlines, legend gone for the single series). Self-check: change a revenue number in the workbook, refresh the link, and confirm the column on the slide actually moves. If it doesn't move, you don't have a linked chart yet, whatever the Paste Special dialog told you — go back and check which paste option you actually selected.

The AI moves — climbing the ladder

Same task — get this workbook's chart onto this slide, correctly, and prove it's alive — with more handed to AI at each rung. The check never changes: change the number in the workbook, and the slide must move, or you haven't linked anything.

Rung 2 · Aldi:GPT — it prescribes the paste flavour, you judge the prescription

Aldi:GPT can't see your deck or your workbook, so describe your deck's data slides and how often each one's source changes; it will prescribe linked, embedded, or picture per slide, with reasons you then have to hold it to.

Type this prompt into Aldi:GPT

Check: the signature failure at this rung is one-size prescription — "link everything" or "embed everything" as a blanket rule. A good answer treats each of the five slides differently and names the reason: slide 3 shouldn't be linked to a workbook you don't have open, and slide 5 was never data to begin with, so it stays a picture. Any answer that recommends the same flavour for all five slides has missed the point of the rung — "link everything" is exactly the choice that breaks the deck the moment it leaves the building.

Rung 3 · M365 Copilot — it can create the slide, check what it actually created

Copilot in PowerPoint can build a slide that visualises data you give it, but be precise about what you get back: it produces a real chart object you can edit, not a linked-to-your-workbook object — those are two different things, and only you can tell them apart before the month rolls over.

Type this prompt into Copilot (in PowerPoint, with the deck open)

Check: click the chart Copilot hands you and look at what's selected — a real chart object (with a "Chart Design" ribbon and an editable data table behind it) or a static image dropped onto the slide. Copilot's charts are genuine chart objects, but they are not linked to your source workbook — Copilot built them from data you gave it in the prompt or fed it as a file, which is a snapshot, not a wire back to the workbook. Know which one you got before the month rolls: if it needs to move with the workbook, this chart still needs the manual Paste Special → Paste Link from rung 1, not a Copilot rebuild each cycle.

Rung 4 · GitHub Copilot — it scripts the chart generation, with a refresh path

An agent with file access can write a script that builds deck charts directly from your data files using python-pptx's native chart objects — genuinely reproducible, but only as good as the refresh path you make it prove.

Type this prompt into GitHub Copilot

Check: rerun the script after changing a number in the source CSV — the chart on the slide must move to match, exactly the same proof as rung 1's manual test, just automated. Then check the reconciliation, the family discipline this course shares with the Excel course: the totals on the slide must sum back to a plain total of the source file, parts summing to the whole. A script that regenerates a chart but silently drops or double-counts a category has failed the check even if the chart looks clean.

The point of the ladder From "AI prescribes linked vs embedded vs picture" to "AI scripts the charts straight from the data file" — the deliverable never changes: a chart on a slide that stays honest to its source, and a paste flavour chosen on purpose rather than by default. What protects you at every rung is the same test — change the number, watch the slide move, or admit it didn't. Craft the manual paste-link first, so you can tell a real link from a chart object that only looks like one.
The family discipline This is the same discipline the Excel course's sourcing lessons taught about the workbook, and the Outlook course's send-mechanics lesson made the identical point about email: data has to travel as data, not as a picture of data. A screenshot pasted into an email and a screenshot pasted into a slide fail for the same reason — the receiver, or the deck itself, is left holding an image nobody can act on or verify.
