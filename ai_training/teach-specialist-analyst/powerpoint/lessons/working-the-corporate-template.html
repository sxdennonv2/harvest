PowerPoint to a High Standard · Lesson 0007

Working the corporate template (not against it)

Tied to your mission: turning analysis into decks that carry the finding, not decorate it, means the brand and layout decisions shouldn't be yours to remake every time — someone already made them, and they live in the corporate template. New decks born from that template arrive looking right for free. Decks born from Save-As on last quarter's file carry forward every fix, every stale footer, every off-brand hex that quarter's deck ever picked up — and that's exactly how the franken-deck from Masters & layouts got three masters riding in one file. This lesson is about not making that mistake on Thursday.

Practice pack · open this first

Open the practice pack and find Scenario D — The template job: a supplier-day deck needed by Thursday, the corporate template sitting right there, and the usual shortcut — copy last quarter's deck, overtype it — waiting to be taken instead.

The one idea

A corporate template is a contract, not a suggestion. Its layouts, its theme colours, and its theme fonts encode decisions someone already made — which typeface reads best at distance, which blue is actually the brand's blue, which arrangement of placeholders suits a section break versus a comparison slide — so that you don't remake those decisions, slightly differently, every single time you build a deck.

Working with the template means three habits: start every new deck with New-from-template, never Save-As on an old file; treat the layout gallery as your menu — section header, content, comparison, whatever the job calls for, already built; and pick colours by their role in the theme (Accent 1 means something specific to this brand) rather than by eyedropper off a supplier's logo. Fighting the template — a hand-drawn title box, an off-palette hex typed into the fill dialog, a logo dragged 10% larger because it "looked small" — costs you an hour now, and it costs the brand every time that slide gets copied into someone else's deck afterwards. That's the franken-deck's origin story, one shortcut at a time.

Why it matters to a specialist analyst

Supplier-day decks are exactly the kind of deck that gets built under deadline pressure with a "just get it done" mindset — a category walkthrough for Thursday, stitched together from whatever's fastest. The fastest wrong move is opening last quarter's supplier-day deck, Save-As, and overtyping the supplier names. That file is already carrying last quarter's fixes, last quarter's stale footer date, and whatever off-brand hex someone typed in for a supplier's logo colour six months ago. None of that is visible on Tuesday. All of it is visible when the director flips through it on Thursday and asks why this deck looks different from the one procurement presented last week.

Your reporting cycle makes this worse than a one-off embarrassment: sustainability metrics and category decks recur every quarter, so a template violation you let slide once becomes the new "normal" the next analyst copies from — sediment, exactly like the source data errors you'd never let slide in a SAP extract. The specialist analysts who don't live this way aren't better at PowerPoint — they just never start from an old deck. They start from the template every time, so the only thing that changes deck to deck is the content, and the brand never drifts.

The bad habit this undoes Save-As inheritance: every new deck is a copy of the last one, so every new deck also inherits the last one's accumulated patches, its stale footer, and its off-brand hexes — sediment, layered deck after deck, until nobody can say what the template actually specifies any more. New-from-template is the only move that starts clean.

How to do it

Work Scenario D. First, do not open last quarter's supplier-day deck at all. Start from File → New and choose the corporate template (or open the template file itself and immediately Save As a new deck name — never open an old deck and Save As from there; the difference is which file's history you're inheriting). Confirm you're on the template's master before you type a single word: View → Slide Master should show one master, the corporate one, with its layouts underneath it.

Now build the supplier-day skeleton entirely from the layout gallery — Home → New Slide → pick a layout, never Home → New Slide → blank. A section header layout for each supplier's segment ("BrightHarvest", "Riverside", "GreenField"). A content layout for each supplier's standard talking points (volumes, pricing, issues). A comparison layout — two content columns side by side — for the one slide that puts this quarter's numbers next to last quarter's. Every slide kind the day needs already exists in the gallery; the job is choosing correctly, not inventing.

Next, bring in one slide you're reusing from elsewhere — say, a standard "supplier scorecard" slide you keep reusing across reviews. Paste it with destination formatting (the paste-options icon that appears after pasting, or Ctrl+Alt+V → "Use Destination Theme"), not "Keep Source Formatting." Watch what happens: the pasted slide's fonts and colours snap to this deck's theme. That's the template doing its job — the slide adopts the deck it landed in, instead of dragging its old deck's formatting in with it.

Finally, you'll hit a moment where the deck wants a colour the theme doesn't have — a supplier's brand colour for a callout box, say. Do not open the fill dialog and type a hex code. Instead, write the sentence "that's a brand question, not a formatting one" in your notes or the deck's comments, and raise it with whoever owns the template. A colour outside the theme is a decision for the template's owner, not a one-off you make alone at 4pm on Wednesday.

Try it for real

Start a real deck you actually need this week using File → New from your organisation's template — not a copy of an old file, even one of your own. Build its skeleton entirely from the layout gallery, and if you reuse one slide from an existing deck, paste it with destination formatting. Self-check: open View → Slide Master when you're done. You should see exactly one master, every slide should show a real claimed layout (not "Custom Layout"), and every colour on every slide should trace back to a named theme colour — if you can't say which theme colour a fill is, by role, it probably isn't one, and that's the tell you fought the template somewhere.

The AI moves — climbing the ladder

Same task — start a new deck from the template and build its skeleton on the layout gallery — with more handed to AI at each rung. The check never changes: every slide traces to a real layout from the one template master, and every colour traces to a named theme role, never a one-off hex.

Rung 2 · Aldi:GPT — it maps sections to layouts, you judge the map

Aldi:GPT can't see your template file, so it can only work from what you describe; you judge whether the layout it names for each section actually exists in your gallery and actually fits the job.

Every mapping in its answer must land on one of the layouts you listed; if it can't find a fit, it must say so as a gap for the template's owner to fill, not invent a workaround layout of its own.

Rung 3 · M365 Copilot — it builds from the template, it doesn't replace it

Copilot in PowerPoint can create a new deck and populate a section skeleton from a prompt, but it only stays on-template if you open the template file first — ask it to "create a presentation" from a blank start and it will bring its own design ideas, not your brand's.

Type this prompt into Copilot (in PowerPoint, with the template file open)

Check: open Slide Master view on the result. Copilot must have built on the template's existing master and layouts — one master, every new slide showing a claimed layout from it. If you see a second master, a "Custom Layout," or slide designs that don't match your theme, Copilot brought its own design instead of using yours; re-prompt with the template file open first and say so explicitly rather than accepting the off-template result.

Rung 4 · GitHub Copilot — it builds the template itself, from your brand spec

An agent with file access can generate the template file itself via python-pptx — master and core layouts — from a written brand spec, but it can't infer your brand from nothing; you hand it the colours, fonts, and logo rules explicitly.

Type this prompt into GitHub Copilot

Check: the test is whether the colours are real theme colours, not paint. Change Accent 1 in the brand spec to a different blue and regenerate — every shape using Accent 1 across every layout must update automatically. If a colour only changes on some shapes, or you find hard-coded RGB values in the script standing in for theme roles, the template is a painting of a template, not a contract — send it back and ask for role-based theme colours, inheriting, not per-shape fills.

The point of the ladder From "AI maps sections onto existing layouts" to "AI builds the template itself from a brand spec" — the deliverable never changes: a deck (or a template) whose layouts and colours are used and defined by role, not invented or hard-coded slide by slide. What protects you at every rung is that same check, not the tool. Learn to read a template's contract by hand first, so you can tell a real layout-gallery fit from a custom workaround, and a theme colour from a one-off hex — whether it's you reaching for the eyedropper or the AI reaching for RGB.
