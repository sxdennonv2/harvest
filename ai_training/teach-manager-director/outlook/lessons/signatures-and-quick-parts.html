Outlook to a High Standard · Lesson 0012

Signatures & Quick Parts: never type it twice

Tied to your mission: the survey's productivity demand was diffuse, but it hides a very literal arithmetic. Type the same three-sentence caveat, the same delivery address, the same sign-off, by hand, fifty-two Fridays a year, and the small accidental variations start to matter — the week the caveat drifts, it drifts on a document a supplier can hold you to. Signatures and Quick Parts are the two building blocks that stop the retyping: set once, inserted identically, every time.

Practice pack · open this first

Open the practice pack and read Scenario E — The Friday ritual again. Its acknowledgement text — "Received, will review Monday — flag anything unusual now." — is the specimen you'll save as a Quick Part in this lesson.

The one idea

If you've typed it twice, it's a building block. Outlook gives you two kinds, and they carry different things. Signatures carry identity — who you are, applied automatically to every message you send, so you set it once instead of signing off by hand. Quick Parts carry content — reusable blocks of text you save once and insert by typing the block's name, so a paragraph you send weekly is never retyped, and never quietly drifts.

A signature isn't one thing, it's a pair: a full version for new mail (name, role, contact — what a stranger needs to reach you) and a lean version for replies (name only — a six-message thread doesn't need six logo banners marching down the page). The six-banner reply chain you've seen isn't a formatting accident; it's a signature configured wrong. And this is a different instrument from Quick Steps: Quick Steps chain actions — move, flag, reply, one click. Quick Parts store content — the words themselves. You'll often use both on the same message.

Why it matters to a manager

Some of what you retype is legally load-bearing, not just repetitive. The standard samples caveat — the line that tells a supplier a sample price isn't a committed order price — has to be identical every time it's sent, because "identical" is what makes it enforceable as a standing position rather than a one-off remark a supplier can argue was different last time. You didn't necessarily draft that caveat's wording yourself, but you commission and review the mail that carries it, and you're the one a supplier dispute lands on. Retype it from memory fifty-two times a year and it acquires small accidental variations: a dropped clause here, a softened word there. Each variant is a slightly different caveat, and the one time it matters is the one time it's wrong — the one GDM or a senior audience asks you to stand behind. A Quick Part removes the retyping, which removes the drift — the caveat that goes out this Friday is character-for-character the one that went out last Friday, because it's the same saved block, not a memory of it. The same discipline pays off in every meeting-heavy week: the sign-off, the delivery block, the acknowledgement you send between meetings shouldn't cost you a rewrite each time.

The bad habit this undoes Two flavours of the same mistake, and most inboxes carry both. The first is retyping the same three sentences — the caveat, the delivery address, the acknowledgement — from memory, fifty-two times a year, with small variations each time that nobody notices until a supplier reads two versions side by side. The second is signature soup: no signature at all, then someone sets up three slightly-different ones on different devices, then a reply chain that's been replied-to six times is carrying six copies of a logo banner nobody meant to send six times. Both are the same root failure — treating a repeated block as something to retype or reapply by hand instead of something to save once.

How to do it

Build the proper signature pair, then save Scenario E's acknowledgement as a Quick Part.

Set up the signature pair. In Outlook, go to File → Options → Mail → Signatures (or, in new Outlook, Settings → Mail → Compose and reply). Create a new signature named Full — new mail and give it your name, role, and one line of contact detail (phone or extension) — the minimum a stranger needs to know who you are and how to reach you. No quotes, no banners, no inline logo image that arrives at the far end as a stray attachment. Create a second signature named Lean — reply containing your name alone. In the two dropdowns below the signature list, set New messages to the full signature and Replies/forwards to the lean one. This pairing is the whole fix for the six-banner reply chain: the identity block shrinks to a name once the thread is already established.

Save the acknowledgement as a Quick Part. Open a new message and type the acknowledgement text exactly as the practice pack states it: Received, will review Monday — flag anything unusual now. Select that text, then go to Insert → Quick Parts → Save Selection to Quick Part Gallery. Name it something you'll recognise at a glance — Friday ack works — and click OK. Delete the typed text from the draft (it's saved now; you don't need the copy in front of you).

Insert it by name. In a fresh draft, click where the acknowledgement belongs, start typing the Quick Part's name — Friday ack — and press Enter when Outlook's autocomplete offers it (or go back through Insert → Quick Parts and pick it from the gallery). Check the inserted text against the pack's wording character for character: it must be the same block every time, which is the entire point.

Try it for real

Search your own Sent Items for a phrase you know you've typed more than once this month — the search skill you already have from this course, turned on your own writing instead of someone else's mail. Find the paragraph you've retyped the most, select it in a message where it appears, and save it as a Quick Part with a name you'll actually remember. Then use it once for real: start the next message where that paragraph belongs, type the block's name, and insert it instead of retyping. Check: the block inserts verbatim — no re-keying, no chance of a small accidental variation — and you can say out loud which instrument you used and why: a Quick Part, because you were saving reusable content, not chaining actions (that would have been a Quick Step).

The AI moves — climbing the ladder

Same task — build the identity and content blocks that stop the retyping — with more handed to AI at each rung. The check never changes: does every block trace to something you actually, really send, and does the load-bearing line inside it (the caveat, the "flag anything unusual now" ask) survive whatever polish the tool adds?

Rung 2 · Aldi:GPT — it drafts the block library, you verify it's really yours

Aldi:GPT can't see your mailbox or your Sent Items, so it can only draft from what you describe — what you retype, to whom, and why. You supply the real recurring text; it organises it into a signature plus a small set of named Quick Parts.

Every block must trace back to text you really send; the signature must carry only what a receiver needs — name, role, contact — nothing decorative.

Rung 3 · M365 Copilot — it drafts or refines a block's text in context, it cannot save it as a signature or Quick Part

Be honest about the boundary: Copilot in Outlook cannot create a signature or save a Quick Part — there is no "install this block" button it can press. Its real rung-3 move is drafting or tightening a block's wording in the compose window before you select it and save it yourself.

Rung 4 · GitHub Copilot — it builds the identity kit as files, it cannot install anything into Outlook

GitHub Copilot is an agent with file access but no connection to your live Outlook profile — it cannot create a signature or a Quick Part for you. What it can do, from a plain-text description of your blocks, is build the kit around them: the signature as clean minimal HTML that Outlook will actually accept, each Quick Part's text as its own file, and a setup card listing the exact click-path to install each one by hand.

And the signature HTML must be genuinely minimal: a kit that ships a 40-line table-soup signature has automated the wrong thing — the goal was one clean identity block, not a bigger version of the mess this lesson undoes.

The point of the ladder From "AI drafts the block library" to "AI builds the identity kit as files" — the deliverable at every rung is the same signature pair and the same small set of Quick Parts, saved into Outlook by a human hand, because none of these tools can reach into your mailbox and click Save for you. What protects you is the same check throughout: does each block trace to something you really send, and does the load-bearing line inside it survive. Craft first, so you can supervise.
