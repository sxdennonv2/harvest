Word to a High Standard · Lesson 0006

Figures & captions that stay where you put them

Tied to your mission: a range-review submission carries at least one chart, and the difference between a chart that reads as evidence and a chart that reads as an accident is entirely decisions you made — or didn't — when you dropped it in. This lesson closes the tables theme: tables that behave taught the grid to hold its shape; this one teaches the picture beside it to hold its place, and its number to stay true.

Practice pack · open this first

Open the practice pack and find Scenario C — The drifting table: its chart, pasted from the course workbook, has floated to the wrong page, and its caption was typed as plain text — "Figure 3:" typed by hand, not generated by anything.

The one idea

A figure in Word does one of two things, and both are decisions: it flows, or it floats. In-line with text, a figure behaves like one big character in its paragraph — it moves exactly where that paragraph moves, page after page, edit after edit, and it cannot wander, because it isn't a separate object sitting on top of the page; it's part of the text flow. That's the right choice for most report figures, this course's chart included, because a report figure's whole job is to sit where the prose that discusses it sits. A wrapped figure floats instead — it becomes an object anchored to a paragraph, free to sit beside or between text, and the anchor is the whole story: the figure follows its anchor paragraph, wherever that paragraph goes. Scenario C's chart wandered because nobody chose its anchor on purpose — it drifted along with whatever paragraph happened to own it, onto whatever page that paragraph landed on next.

A caption is the second half of this idea, and it has the identical shape: a caption is either a field or it's typed text pretending to be one. Insert Caption writes a real, numbered field — Word counts every captioned figure in the document and keeps that number current, so adding a figure earlier in the document renumbers everything after it automatically, and a table of figures can be built from those fields the same way a table of contents is built from headings. A typed "Figure 3:" is just a string of characters. It is correct exactly once, at the moment you typed it, and wrong the first time anyone inserts a figure before it.

Why it matters to a buyer

The chart in a range-review submission is usually the one thing a director looks at before reading a word of the surrounding prose — it has to be exactly where the paragraph discussing it says it is, not two pages later because an edit upstream pushed a paragraph down. And submissions get figures added late: Merch asks for one more comparison chart the night before the deadline, and if every caption before it was typed by hand, that one insertion means renumbering every "Figure" reference in the document by hand too, including any that got quoted in the prose ("see Figure 4") — the kind of small, boring correction that's exactly the kind that gets missed under deadline pressure, and shows up as a submission that cites its own figures wrong.

The bad habit this undoes Paste-and-nudge: drop the chart in with whatever wrapping Word defaults to, drag it "into place" by eye, and watch it teleport two pages away the next time anything above it changes — because dragging a floating object never chose its anchor, it just repositioned the object relative to wherever its anchor already was. Alongside it, captions typed as plain text and renumbered by hand every time a figure is added or removed, silently wrong the moment anyone forgets a step — and nobody forgets loudly; the document just quietly starts lying about its own figure numbers.

How to do it

Open Scenario C and find the wandering chart. First, decide flow vs float on purpose, not by whatever Word defaulted to when it was pasted. This is a report figure discussed by the surrounding prose — it should read as part of that paragraph's argument, not as a decoration beside it — so it belongs in-line with text, in its own paragraph. Click the chart, open Layout Options (the icon that appears beside a selected image or object), and choose In Line with Text. Once that's set, the chart behaves like a character: press Enter above it, and it moves down with everything else. It cannot float away from that paragraph, because it no longer has an anchor to float on — it is the text.

If a figure in your own work ever genuinely needs to float — a small illustrative image wrapped beside a paragraph, say — the anchor is what you must look at before you touch the wrapping. Turn on Home → Show/Hide ¶ (or View → Show → the same toggle) so paragraph marks and object anchors are visible; a small anchor icon appears next to whichever paragraph currently owns the floating object. Click the object and drag that anchor icon itself — not the picture — to the paragraph you actually want it tied to, then set Layout Options → Fix position on page if you want it to stay put even when text above it reflows, or leave it "Move with text" if you want it to travel with its anchor paragraph on purpose. Either way, that choice — which paragraph owns this figure, and does it travel with that paragraph or stay fixed — is the decision paste-and-nudge skipped.

Second, fix the caption. Delete the typed "Figure 3:" text entirely — it is not a caption, it only looks like one. Click the chart, go to References → Insert Caption (or right-click the chart and choose Insert Caption), confirm the label is Figure, and click OK. Word writes a real numbered field below the chart — "Figure 1" (or whatever number is correct given the figures before it in the document) — followed by wherever you type the descriptive text ("Supplier price comparison, Q3"). That number is not typed; it is calculated every time the document's fields are updated.

Third, prove it: add a second figure earlier in the document — any image or chart, captioned the same way with Insert Caption — then select the whole document (Ctrl+A) and press F9 to update all fields (or right-click each caption and choose Update Field). Watch the original chart's caption change its own number to make room for the new one. Nobody retyped anything; the field recalculated itself, the same way a formula recalculates when a cell upstream changes.

The chart itself, worth noting in passing since it arrived from Excel: how it got onto this page in the first place is a different seam with its own discipline — the paste-link habit the PowerPoint course teaches for Excel-to-slide charts applies here too. A chart pasted as a picture is dead the moment the workbook changes; whether it should be pasted live is a decision this lesson doesn't own, but it is a decision, not a default.

Try it for real

On Scenario C's chart, do the full fix: set it to In Line with Text in its own paragraph, delete the typed caption, and replace it with a real Insert Caption field. Then add a second figure somewhere earlier in the document with its own Insert Caption, and update fields (Ctrl+A, then F9). Self-check: the original chart's caption number must have changed on its own to stay correct, and pressing Enter above the chart's paragraph must move the whole figure down with the text — it should be structurally impossible for it to end up on a page disconnected from the paragraph that discusses it.

The AI moves — climbing the ladder

Same task — get a figure to stay where its discussion is, and its caption number honest — with more handed to AI at each rung. The check never changes: a figure that cannot structurally wander from its paragraph, and a caption number that updates itself rather than being typed.

Rung 2 · Aldi:GPT — it prescribes flow vs float per figure, you judge the prescription

Aldi:GPT can't see your document, so describe each figure's job in the submission; it will prescribe in-line flow or floating wrap per figure, with the anchor paragraph named for any it recommends floating.

Type this prompt into Aldi:GPT

Check: the signature failure at this rung is float-by-default prescriptions — recommending wrapping because "it looks nicer floating." Push back on any recommendation that doesn't justify why in-line flow specifically fails for that figure. A good answer keeps the price-comparison chart and the floor-plan diagram in-line (both are evidence a paragraph is actively discussing), and only floats the decorative logo, with a stated anchor. "It looks nicer floating" is how the wandering starts — every float in the answer must earn its place.

Rung 3 · M365 Copilot — it can draft the prose that cites the figure, not the figure itself

Be honest about the boundary: wrapping, anchors, and Insert Caption are native Word, not a Copilot capability. What Copilot can do is draft or tighten the prose that refers to a figure by number — and that's exactly where it can go wrong, because it reads the words on the page, not the caption field behind them.

Type this prompt into Copilot (in Word, with the document open)

Check: click the actual caption field beneath the figure and read its real, current number — then compare that against whatever number Copilot's sentence used. Copilot reads the visible text on the page, not the field's live value, so if a typed caption nearby ever says "Figure 3" while the real field-driven number is now "Figure 2," Copilot will happily write "as Figure 3 shows" and cite the typed lie. Trust the field, not the prose, and correct the sentence by hand if they disagree.

Rung 4 · GitHub Copilot — it audits every figure's wrapping, anchor, and caption type

An agent with file access can write a script that inspects a .docx and reports, per image, exactly what this lesson cares about: whether it's in-line or floating, which paragraph anchors it, and whether its caption is a real field or typed text impersonating one.

Type this prompt into GitHub Copilot

Check: run it against Scenario C's original spec first — it must flag the wandering chart's float and the typed caption, both. Then run it against your tamed version — it must come back clean: in-line, no anchor concern, caption is a real field. Watch for the failure mode on the other side too: an auditor that flags every floating figure, with no attempt to read whether the anchor makes sense, isn't auditing your document — it's enforcing a blanket taste preference and calling it a bug report.

The point of the ladder From "AI prescribes flow vs float per figure" to "AI audits wrapping, anchors, and caption type across the whole document" — the deliverable never changes: a figure that cannot structurally wander from the paragraph discussing it, and a caption number that recalculates itself instead of being typed and forgotten. What protects you at every rung is the same two facts — In Line with Text can't drift, and Insert Caption can't lie. Learn to set both by hand first, so you can tell a justified float from a lazy one and a real caption field from a convincing fake at every rung above it.
