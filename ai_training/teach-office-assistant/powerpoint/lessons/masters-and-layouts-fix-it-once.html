Masters & layouts: fix it once, fix it everywhere

Tied to your mission: fixing layout, alignment and branding fast — masters and layouts, not nudging boxes — starts here, before alignment, before charts, before any of it. Much of what you produce goes out under the department's name, so a deck's skeleton — the master and its layouts, the part you rarely look at — is where professional polish actually lives or dies. This lesson is the floor: fix problems at the level that owns them, and the rest of this course stops fighting the same fire on every slide.

Practice pack · open this first

Open the practice pack and find Scenario C — The franken-deck — a range-review deck assembled from three older decks, three slide masters aboard, four title styles, footers that disagree about the date, and a logo that jumps 5mm between slides.

The one idea

A deck has a skeleton you rarely see: the master owns what's everywhere — the fonts, the logo, the footer, the colour theme. A layout (Title Slide, Title and Content, Section Header, and so on) owns the arrangement of placeholders for one kind of slide. A slide only holds content — it inherits everything else from the layout it claims, which in turn inherits from the master.

That gives you a rule for where to fix anything: fix at the highest level that owns the problem. A master-level fix (the logo's position, the footer's date format) lands on every slide in the deck at once. A layout-level fix (how a Title-and-Content slide arranges its placeholders) lands on every slide of that kind. A slide-level fix lands once, on that slide only. Editing slides one at a time for a problem the master owns is fixing one leaf of a tree, forever, instead of fixing the trunk.

Why it matters to an office assistant

Training material and department decks get assembled the same way every cycle: last quarter's deck, a colleague's slides, a template someone downloaded two years ago — all pasted into one file the night before it's due. Each import drags its old master in with it. Nobody notices until the deck goes out under the department's name and the logo jumps, the footer date is wrong on half the slides, and every title looks like it came from a different company.

Keeping department decks and training material to a consistent standard isn't about hand-fixing faster. It's consolidating onto one master before you touch a single slide's content, so the correction — the right logo, the right footer, the right title style — happens once and holds for every deck built from that file afterwards.

The bad habit this undoes The franken-deck workflow: assemble slides from old decks, then hand-fix each slide's logo position, footer date, and title style, one slide at a time, forever — because nobody opened View → Slide Master to see that three masters were riding along in the same file. Every "quick fix" is a patch on a leaf when the problem lives in the trunk.

How to do it

Work Scenario C's franken-deck. First, open View → Slide Master. You'll see more than one master in the left-hand pane — that's the tell. Three masters riding in one file means three sets of fonts, colours, and footers, one per donor deck. Pick the master that matches where this deck actually belongs (the house style, not whichever one happened to be first), and delete the other two from the Slide Master view.

Deleting the extra masters doesn't fix the slides that were built on them — those slides still remember which layout they came from, and PowerPoint will flag or orphan them. Select those imported slides, right-click, and choose Reset, then explicitly assign each one the correct layout from the surviving master (Layout picker, or right-click → Layout). Reset snaps placeholder content back to the layout's arrangement; it does not, by itself, move a slide onto a different master's layout — you still choose the layout.

Now make the corrections once, at the level that owns them. The logo and the footer's date format belong to the master itself — fix them in the master view and they appear correctly on every layout and every slide below it, in one edit. The title style belongs to whichever layout is misbehaving — if only the Title-and-Content layout has an oversized title placeholder, fix that layout, not the fifteen slides using it. Narrate each fix as you make it: "this one's a master fix" or "this one's a layout fix" — if you ever reach for a slide-level fix here, stop and ask which level actually owns it.

Check yourself

Answer these before moving on — immediate feedback below each.

1. Every slide in the deck has the wrong logo, in the same wrong spot. Where's the correct fix?

2. Only the Section Header slides have an oversized title that collides with the subtitle. Every other slide kind is fine. What owns this?

3. You deleted two extra masters and ran Reset on the imported slides, but three of them still show the wrong layout's arrangement. What's the missing step?

Try it for real

Open a deck of your own that's been assembled from more than one source — a training deck stitched from last quarter's, a colleague's slides pasted in, anything with more than one donor. Open View → Slide Master and count the masters. For every problem you can see repeated across slides (a logo, a footer, a title style), name which level owns it before you touch anything. Self-check: after your fixes, you should be able to point at any slide and say which single edit — made once, at the master or the layout — is why it now looks right, with no slide-level patch left standing for a deck-wide problem.

The AI moves — climbing the ladder

Same task — diagnose a franken-deck and fix each problem at the level that owns it — with more handed to AI at each rung. The check never changes: every fix must land at the highest level that actually owns the problem, never patched slide by slide underneath it.

Aldi:GPT can't see your deck, so it can only work from what you describe; you judge whether its master/layout/slide mapping is actually correct.

Check: the signature failure at this rung is level confusion — a fix prescribed at slide level for something the master owns (like problem 1 or 2 above, which should both resolve to "master"). Read its answer back against the one idea: master owns what's everywhere, layout owns arrangement for one kind of slide, slide is content only. If it tells you to fix the logo "on each affected slide," it's wrong — send it back.

Rung 3 · Microsoft 365 Copilot — it builds onto layouts, it doesn't fix your skeleton

Copilot in PowerPoint can create new slides and sections from a prompt, placing them onto the layouts that already exist in your deck — but it builds on whatever skeleton is already there. If your deck is still a franken-deck when you ask, you get franken-slides back.

Type this prompt into Copilot (in PowerPoint)

Check: before you trust the result, open Slide Master view and confirm the new slides claimed real layouts from your surviving, consolidated master — not a stray layout dragged in from one of the old decks. Reset doesn't move a slide onto a different master's layout, and neither does Copilot; it builds on what's there. If you ran this prompt before consolidating per the manual-craft steps above, expect the new section to inherit the same three-master mess — consolidate first, then let Copilot add to a clean skeleton.

Rung 4 · GitHub Copilot — it builds the skeleton from scratch, on one master

An agent with file access can generate an entire deck skeleton via python-pptx from an outline, choosing layouts from a single named master as it goes — but it can't inspect a messy existing deck for you; you still hand it a clean outline and a target master to build against.

Search the script itself for any add_textbox call standing in for a title or body — that's an orphan text box doing a placeholder's job, the same problem Scenario C's hand-drawn title box has, just introduced by code instead of by hand.

The point of the ladder From "AI maps the problem to a level" to "AI builds the whole skeleton on one master" — the deliverable never changes: a deck where every deck-wide fix lives at the level that owns it. What protects you at every rung is that same check, not the tool. Learn to read a deck's skeleton by hand first, so you can tell a real master fix from a slide-level patch dressed up as one — by you or by the AI.
