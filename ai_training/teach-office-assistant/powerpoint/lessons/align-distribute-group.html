PowerPoint to a High Standard · Lesson 0003

Align, distribute, group: the tidy slide

Tied to your mission: you fix layout, alignment and branding fast for the department's decks — masters and layouts, not nudging boxes. And a slide either measures as tidy or it doesn't. Eyes catch a 2mm misalignment before anyone reads a word on the slide, which means an untidy category-review slide undermines the numbers on it before the room has read a single one — and that slide carries the department's name. This lesson replaces "looks about right" with three commands that make tidy a fact, not a feeling.

Practice pack · open this first

Open the practice pack and find Scenario A — The untidy slide — a monthly category-review slide assembled in a hurry, with the correct tidy state given underneath so you can mark your own fix.

The one idea

Tidy is measurable, not felt. A slide is tidy when its objects share common edges (Align), sit at computed equal gaps rather than eyeballed ones (Distribute), and move as single units where they belong together (Group). All three are commands, not adjustments — you select the objects and let PowerPoint do the geometry, because a ruler beats an eyeball every time.

The habit this replaces is nudging boxes with the arrow keys until they "look about right". They never are — and Scenario A is exactly what "looks about right" produces when nobody reached for the instruments instead.

Why it matters to an office assistant

The monthly category review is a recurring deck with recurring stat boxes — the kind of slide you rebuild under time pressure every reporting cycle, and it goes out under the department's name, so it has to hold up to a scan. Someone reading three numbers side by side registers a 2mm height mismatch as "sloppy" before they register which number is highest, and that reaction lands on the department's polish, not the formatting. A slide fixed with Align and Distribute stays fixed the next time you touch it, because the objects that belong together are grouped — one edit, one move, nothing drifts apart again, which matters when this is the deck you'll be asked to refresh every month.

The bad habit this undoes Nudging a box with the arrow keys while squinting at the one next to it — five minutes spent on a single box, and it's still 2mm out because eyes are bad rulers. Then the next edit to the slide breaks the "fix" again, because nothing was grouped, so moving one box for the title never took its neighbours with it.

How to do it

Open Scenario A's slide (reconstruct it from the pack's spec if you're not rebuilding it live) and fix it with commands only — zero arrow keys, start to finish.

Select the three stat boxes. On the Shape Format (or Home) ribbon, open Arrange > Align and apply Align Top — the left box that sat 2cm high now shares the same top edge as the other two. With all three still selected, reopen Align and apply Distribute Horizontally — the uneven 1.1cm/2.6cm gaps become one equal gap, computed from the boxes' combined width, not guessed between them.

Next, select the stat boxes together with anything that labels or frames them as a set and apply Group (Arrange > Group > Group, or Ctrl+G) — from here they move, resize, and reflow as one object, so the alignment you just set can't be knocked apart by a later edit to just one of them.

Click the rotated fourth box and clear its rotation: open the Format Shape pane and set Rotation to 0° — a stray 1° with no explanation is a decision nobody made, not a design choice, so zero it. Select the pasted-Calibri survivor and reset its font to the template's font from the Home ribbon's font box — one font family on the slide, matching the master. Finally, drag the title back into its own placeholder (or use Reset on the slide if it's sitting in a stray text box) so it clears the logo instead of overlapping it.

Check yourself

Answer these before moving on — immediate feedback below each.

1. Three stat boxes need level tops and equal horizontal gaps. What's the command sequence?

2. Why group the stat boxes after aligning and distributing them?

3. A box sits rotated 1° with no stated reason. What makes it "tidy"?

Try it for real

Take a slide of your own with more than two objects that should share an edge or a gap and fix it using only Align, Distribute, and Group — no arrow keys. Self-check against Scenario A's correct-tidy box: every box that should share an edge does, gaps between siblings are equal (not just close), things that move together are grouped, no rotation survives that you didn't choose on purpose, and text lives in its placeholder, not floating over something else.

The AI moves — climbing the ladder

Same task — take an untidy slide to the state Scenario A's correct-tidy box describes — with more handed to AI at each rung. The check never changes: every fix must resolve to a named, executable command (Align/Distribute/Group/Rotation-to-0/font-reset), and nothing that was content before the fix is missing or demoted after it.

Rung 2 · Aldi:GPT — it names the fixes, you execute them

Aldi:GPT can't see your slide, so it can only work from what you describe; you carry out every command yourself and judge whether its list is actually executable.

Check: the signature failure at this rung is aesthetic hand-waving — "balance the composition" or "make it feel more even" instead of a command. Reject any fix you can't immediately go and click; every line in the answer must name an executable operation (Align, Distribute, Group, a rotation value, a font name).

Rung 3 · Microsoft 365 Copilot / Designer — it rearranges, you audit

Designer can act on the open slide and propose a rearranged layout directly, but it doesn't know which elements are supposed to travel together or which text is load-bearing — that judgement stays with you.

Apply Designer to the untidy slide, then check

Check: Designer rearranges but must be audited, never accepted blind. Confirm the three stat boxes are still together as a set (not split across the layout), confirm no text was cropped or quietly demoted to a decorative image, and confirm the title is readable and unobscured. Accept the layout change; never silently accept a content change.

Rung 4 · GitHub Copilot — it audits the deck's geometry by script

An agent with file access can write a python-pptx script that measures a deck's geometry and reports problems — it can't fix your judgement calls, only surface the measurable ones.

Type this prompt into GitHub Copilot

Check: run it on Scenario A's spec first — it must return at least three findings (the near-miss top edges, the uneven gaps, the rotation). Then run it on your tidied version — it must come back silent. A geometry auditor that fires false positives on a deliberately asymmetric layout is measuring taste, not alignment; if it flags a gap or edge you set on purpose, that's the script's bug to fix, not your slide's.

The point of the ladder From "AI names the fixes" to "AI audits the geometry by script" — the deliverable never changes: a slide where every claimed edge, gap, and rotation is a decision you can name, not a guess. What protects you at every rung is the same instrument-based check, not the tool. Learn to align, distribute, and group by hand first, so you can tell a real fix from aesthetic hand-waving and a genuine geometry finding from a false positive.
