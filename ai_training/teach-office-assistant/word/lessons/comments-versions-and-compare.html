Word to a High Standard · Lesson 0008

Comments, versions & compare

Tied to your mission: a range-review submission doesn't move through one reviewer, it moves through several, and Merch's comments arrive alongside your director's tracked changes on the same draft. Track changes without fear gave you the discipline for edits you can see move on the page — accept or reject, verified against the source, never Accept All. Comments are a different animal: they don't change a word by themselves, so nothing forces you to deal with them, and a document can look "reviewed" while its margins are full of questions nobody closed. This lesson closes that gap, and adds the one you can't see coming — the colleague who sends back a document with tracked changes switched off, so the only way to know what moved is to ask Word to compare.

The one idea

Every comment closes with a disposition, and every mystery version has a diff. A comment is an ask wearing a margin, and the one-ask discipline you'd apply to an email applies here too: each one gets resolved (you did the thing, in the document, and the comment can go), answered (you replied in-thread with the decision and that's the whole job), or spawned (the real work is outside the document, so a new instrument — an email, a flag, a task — carries it, and only then does the comment resolve). Mixing these up is how margins rot: Merch's "check this with Supply Chain" cannot die inside a comment thread just because you clicked Resolve on it.

Versions have their own version of the same discipline. When a colleague hands back "final_v3_amended.docx" with tracked changes switched off, you have no record of what they touched — only their word for it. Review → Compare rebuilds the conversation after the fact: put the original and their return side by side, and Word turns every silent difference into a tracked change you adjudicate with the exact same proof-pass discipline you already use — a changed figure gets verified against the source, not nodded through because it arrived looking finished.

Why it matters to an office assistant

A department document you've circulated for review often comes back carrying two different kinds of markup at once: your director's tracked edits, and a scattering of comments from whoever else had a stake in it — a question about a date, a style quibble on a phrase, a "check this with Facilities" that needs someone outside Word entirely before the document can go out under the department's name. If those comments sit half-answered, the document looks reviewed on the surface while a live question is still open underneath it, and the failure only surfaces when someone downstream assumes silence meant agreement — awkward when it's minutes or a procedure with your department's name on it. And when a colleague hands a draft back with tracked changes switched off "because I only touched formatting, honest," the only way to find the one detail they also nudged is to compare, not to trust the claim. An office assistant who skips Compare on a returned draft is trusting the same kind of silent edit that Track changes without fear taught you never to accept unverified.

The bad habit this undoes Comment archaeology: a margin full of half-answered questions nobody formally closed, and — worse — a "check with Supply Chain" ask that got clicked Resolved the moment someone half-remembered mentioning it in a corridor, so the actual follow-up never happened and never will, because the only record of it was a comment thread that no longer shows as open. Its twin habit: taking a returned draft's "I only changed formatting, honest" at face value instead of running Compare — the one time that's false is the time nobody checks.

How to do it

Open Scenario D. Your director's tracked changes (exec summary tightened, the volume figure 110 → 100, one paragraph deleted) are the material of Track changes without fear — handle those with that lesson's discipline first. This lesson is about the five comments Merch left alongside them, and the version check that follows.

Work the Review pane top to bottom and give each of Merch's five comments its disposition. The two questions — say, "is this volume figure before or after the Q3 promo?" and "does this supplier still hold the same tier?" — get answered in-thread: type the factual answer as a reply, directly under Merch's comment, then click Resolve. The thread now carries its own answer; nobody has to go hunting for it later. The two style quibbles — wording Merch didn't like, a phrase they'd cut — get a decision, not a debate: reply with what you did ("shortened per your note" or "kept as-is, it matches the exec summary's phrasing above"), make the edit in the text if you're taking it, then Resolve. A style quibble with no decision recorded is the same rot as an unanswered question — it just looks smaller.

The fifth comment, "check this with Supply Chain," is the one that fails differently, and it is the whole point of this lesson. Nothing you can type into that comment's reply box closes it truthfully, because the actual work — asking Supply Chain — happens outside the document. Do not resolve it yet. First, spawn the real follow-up: send Supply Chain a short email now (or, if your inbox triage from the Outlook course is set up, flag it there with a named due date), quoting the exact figure or clause Merch flagged. Reply to Merch's comment naming where the ask went — "sent to Supply Chain, [date], awaiting reply" — and only then click Resolve. The comment is closed because a real instrument now owns the follow-up, not because the margin went quiet.

Now the version check. Suppose Merch's edits had instead come back as a whole replacement file, "range-review_v3_merch.docx," with no tracked changes visible — just a clean document and a claim that only formatting changed. Open your original alongside it, go to Review → Compare → Compare, set the original document as one side and Merch's return as the other, and let Word generate the comparison. Word produces a new document showing every difference as a tracked change — insertions, deletions, the lot — regardless of whether the sender tracked anything themselves. Read that generated markup exactly the way Track changes without fear taught: any changed figure gets checked against source before you accept it, on the same footing as if your director had tracked it directly. Compare doesn't trust the claim "just formatting" — it shows you the fact.

Try it for real

Take a document of your own with open reviewer comments — or, if none is handy, a draft you can annotate with a few questions and quibbles of your own to practise on. Work every comment to a named disposition: answered in-thread, resolved with a recorded decision, or spawned to a real instrument outside the document before it resolves. Then take (or simulate) a returned version of the same document and run Review → Compare against your original. Self-check: open the Review pane and confirm zero comments remain that are neither answered, resolved-with-a-decision, nor spawned-and-tracked; then confirm the Compare output shows every actual difference between the two versions as a tracked change you can accept or reject one at a time — not a document you took on trust.

The AI moves — climbing the ladder

Same task — give every comment its disposition, and rebuild any silent version's diff — with more handed to AI at each rung. The check never changes: any comment whose real ask lives outside the document must come back as a named spawn, never a quiet Resolve, and any "just formatting" claim gets compared, not trusted.

Rung 3 · M365 Copilot in Word — it drafts replies in context, the buttons stay yours

Copilot in Word can read the open comments and draft reply text in place, but be honest about the boundary: it cannot click Resolve, and it cannot send an email or raise a flag — those actions, and the judgement call on which disposition applies, are yours.

Rung 4 · GitHub Copilot — it builds the reporting tool, it can't touch your document

An agent with file access can write a script against the .docx file itself to report on comment health and to diff two versions paragraph by paragraph — but it cannot open Word, click Resolve, or send anyone an email; you run its report and act on it.

Then run the compare option on a pair of files where one has a single silent fact edit buried among formatting changes — the report must surface that one factual difference by name, not bury it in noise, because surfacing the fact nobody flagged is the entire reason Compare exists.

The point of the ladder From "AI triages the list" to "AI builds the comment-and-compare reporter" — the deliverable never changes: every comment carries a real disposition, and every claimed-silent version gets its diff checked, not trusted. What protects you at every rung is that same check — no spawn-class comment quietly resolved, no "just formatting" return taken on faith. Learn to do both by hand first, so you can catch a tool folding a spawn into a resolve, or a compare report that missed the one fact that mattered.
